# 树枝漫剧

## Overview

树枝漫剧是一套中文 AI 视频生产方法：把长文案拆成连续 15 秒 Board，生成影视化提示词，用上一段真实尾帧驱动下一段首帧，并在多段生成中保持声线、风格、剧情和交接连续。

核心原则：旁白决定画面，真实尾帧决定下一段开场。

## When To Use

使用本 Skill 处理：

- 中文 AI 漫剧、科幻科普、国风短片、纪录式短片、剧情分镜。
- 需要把一篇长文案拆成多条 15 秒以内视频。
- 需要 Seedance/Jimeng/即梦 全能参考类提示词。
- 需要每个 Board 尾帧接下一个 Board 首帧。
- 需要在长队列或多 Agent 之间交接剩余 Board。

若只是单张图提示词、普通文案润色、无连续视频生成需求，不要使用本 Skill。

## Operating Contract

1. **旁白优先**：每条 Board 只承载一个清晰认知点；抽象概念必须变成可见动作。
2. **真实尾帧优先**：有真实视频尾帧时，它覆盖原故事板设想；下一条 0-3 秒必须先桥接真实尾帧。
3. **单并发生成**：付费生成时先查状态，禁止并行提交，禁止重复提交已成功或正在生成的 Board。
4. **双图参考**：第一张图是上一段真实尾帧，作为 0 秒首帧；第二张图是当前故事板图，指导本段内容。
5. **声线一致**：参考音频只用于锁定音色、语速、混响、音效质感和响度；当前 Board 必须按本段旁白重新配音。
6. **随时可交接**：长项目要能打包剩余图片、提示词、起始尾帧、音频参考和说明，交给新会话继续。

## Workflow

1. **读文案与定位**
   - 提取主线：问题/震撼引入 -> 发现 -> 机制 -> 意义 -> 未来畅想。
   - 保留旁白大意，去掉画面中不该出现的现实元素，例如 logo、可读论文页面、品牌标识、未授权人物肖像。

2. **拆 Board**
   - 每个 Board 对应 15 秒以内旁白。
   - 每个 Board 设计 4 个节奏段：0-3 秒承接、3-6 秒揭示、6-11 秒尺度跃迁、11-15 秒锁尾帧。
   - Board N 的尾帧必须可作为 Board N+1 的首帧。

3. **写影视化提示词**
   - 包含：旁白、导演目的、视觉风格、连续性、镜头计划、摄影机与镜头、灯光色彩、声音设计、负面约束。
   - 不要让生成正文包含本地路径、平台参数、费用、归档标题或操作者备注。
   - 需要模板时读取 `references/prompt-skeleton.md`。

4. **生成与巡检**
   - 每次提交前先查队列。
   - 成片后抽首帧、中帧、尾帧。
   - 查看尾帧，判断清晰度、稳定性、主体位置、光线、色彩和是否能接下一段。

5. **修桥接再继续**
   - 若尾帧与下一条规划不一致，先修下一条「连续性」和「0-3秒」。
   - 不要为了迁就旧故事板强行跳切。

6. **长流程压缩**
   - 巡检只保留短输出：Board 编号、队列位置、队列状态、任务标识。
   - 不反复读取完整状态 JSON、长提示词历史或长文档。
   - 会话变卡时读取 `references/handoff.md`，打包交接。

## Visual Taste

默认风格：明媚写实真实世界 + 宏大冷调科幻 + 克制东方宇宙隐喻。

东方元素只作为空间和科学结构隐喻：

- 昆仑轴：高度、世界轴、层间尺度。
- 归墟：收束、深渊、零点。
- 河图洛书：坐标、节点、秩序。
- 玉璧/圭璋：边界、剖面、测量、门。

不要神怪角色、古装堆砌、廉价玄幻、随机发光、PPT 科普感。

## Common Mistakes

- 只追求漂亮画面，忘了旁白关键词。
- 下一条直接按故事板开场，不看上一条真实尾帧。
- 把本地路径、费用说明、平台参数写进生成提示词。
- 参考音频复述旧旁白，而不是重新配当前旁白。
- 会话过长还继续读完整状态和长文档，导致线程卡顿。
